# Branching und mErging

Modell
- Default Branch: main
  
Merges
- Nur per Pull Request in main
- Mindestens 1 Review

Konflikte
- Vor PR aktualisieren: Branch mit main abgleichen
- Konflikte im PR auflösen
- Nach Auflösung erneut prüfen lassen

Releases
- SemVer: MAJOR.MINOR.PATCH
- vX.Y.Z als Git Tag
- Eintrag im Changelog
